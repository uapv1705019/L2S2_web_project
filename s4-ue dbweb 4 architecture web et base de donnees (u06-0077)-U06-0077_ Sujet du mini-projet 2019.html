Mini-projet -- Site e-commerce, compléments

L'objectif du mini-projet est de compléter la version préliminaire d'un site de gestion de tickets d'achats d'utilisateurs, débuté lors du TP 4. Les différentes améliorations proposées sont listées ci-dessous.

Partie 1 - Modélisation des classes

Dans un premier temps un modèle de classe doit être implémenté pour les données.

A faire

  1. A partir des caractéristiques fournies par la base de données, établir le schéma de classe pour le projet.
    Attention : les classes PHP reflètent les objets ISSUS de la base de données pour le programme : i.e. les données telles qu'on les utilisent dans le PHP, pas forcément exactement comme elles sont dans la BD.
    Attention : et donc ces classes devront être conçues en anticipant les modifications du programme demandées dans la Partie 2 (création des pages produits.php et lasttickets.php).
  2. Proposer une visualisation du modèle (qui sera déposé dans un fichier à part lors du rendu). Par exemple avec yuml.me, ou au pire une photo d'un graphe manuel.
  3. Modifier la méthode _toString de chaque classe, afin de régler l'affichage en HTML de chaque classe.

Partie 2 - Intégration PHP/PDO

Il s'agit ici de compléter les fonctionnalités du site initial après avoir adapté le site aux nouvelles classes élaborées ci-dessus.

A faire

  1. Transformer la version du site du TP 4 pour prendre en compte la nouvelle modélisation en classes et récupérer les données de la BD sous forme d'objets (FETCH_CLASS).
  2. Créer une nouvelle page produits.php pour afficher un tableau avec toutes les produits organisés selon leur catégorie. Les produits et les catégories possèdent un lien qui mène à une page lastticket.php pour afficher cette fois-ci les 3 derniers tickets (les plus récents les contenant).
  3. Ajouter les possibilités d'ajout et de modifications des produits : depuis produits.php proposer un modal dans lequel un utilisateur peut saisir les données sur un produit. Si un id valide est donné il s'agira d'une mise à jour, sinon d'un création.
  4. Intégrer une gestion des droits d'accès : maintenant seuls les utilisateurs connectés peuvent voir leurs tickets. Les identifiants sont le nom pour le login et l'id pour le mot de passe. De plus les utilisateurs de la base ont maintenant un statut optionnel 'admin' (vrai ou faux) qui leur permet d'accéder à la visualisation de tous les tickets, ainsi qu'à la lecture et modification des produits.
    Le statut est réglé manuellement dans la base de données.

Partie 3 - Amélioration UX jQuery

Dans cette dernière partie il s'agit de reprendre certaines pages du site pour améliorer l'expérience utilisateur en lui ajoutant des fonctionnalités d'affichage dynamique par le biais de la librairie jQuery.

A faire

  1. Dans la page utilisateurs.php, on ajoute un filtre jQuery sur le tableau qui affiche les utilisateurs, permettant de sélectionner par l'id, le prénom ou le nom.
  2. Dans le page tickets.php, le passage de la souris sur la ligne d'un ticket provoque un zoom automatique pour en faciliter la lecture.

Consignes

Les fonctions du site seront developpées en HTML, CSS, JavaScript et PHP (à l'exclusion de tout autre langage).

La base de données est hébergée sur pedago.univ-avignon.fr.

L'aspect "physique" du site sera réalisé principalement avec W3.CSS. Le site devra être responsive : adapter son affichage en fonction de la taille de l'écran. Pour cela prévoir les 2 cas extremes (grand écran et smartphone) et montrer les modifications introduites en passant d'un cas à l'autre.

Une fois les fonctions précédentes réalisées, tout ajout pourra être envisagé dans la mesure où son apport au site est justifié.
